# Test Suggest コマンド

コードを分析してテストケースを提案するエージェントです。

## 実行手順

1. 対象ファイルまたはディレクトリを特定してください
   - 引数が指定されている場合はそのパスを使用
   - 指定がない場合は最近変更されたファイルを対象

2. コードを読み取り、以下を分析してください：
   - 関数・メソッドの入出力
   - 条件分岐とエッジケース
   - エラーハンドリング
   - 外部依存（API、DB、ファイルシステム）

3. 各関数/メソッドに対して、テストケースを提案してください

## テストケースのカテゴリ

- **正常系**: 期待される入力での動作確認
- **異常系**: エラー入力、境界値、例外処理
- **エッジケース**: null、空配列、最大値/最小値など
- **統合テスト**: 複数コンポーネントの連携

## 出力フォーマット

```
## テストケース提案

### [ファイル名]

#### 関数: `functionName`

**正常系テスト**
1. テストケース名
   - 入力: ...
   - 期待結果: ...
   - 理由: ...

**異常系テスト**
1. テストケース名
   - 入力: ...
   - 期待結果: ...
   - 理由: ...

**エッジケース**
1. テストケース名
   - 入力: ...
   - 期待結果: ...
   - 理由: ...

---
### テスト実装例（オプション）

\`\`\`javascript
describe('functionName', () => {
  it('should ...', () => {
    // テストコード
  });
});
\`\`\`
```

## 使用方法

```
/test-suggest [ファイルパス]
```

引数なしで実行すると、最近変更されたファイルを自動検出します。
