# Simplify コマンド

コードを簡略化・リファクタリングするエージェントです。

## 実行手順

1. 対象ファイルを特定してください
   - 引数が指定されている場合はそのパスを使用
   - 指定がない場合は最近変更されたファイルを対象

2. コードを読み取り、以下の観点で分析してください：
   - 冗長なコード
   - 重複したロジック
   - 複雑すぎる条件分岐
   - 長すぎる関数
   - 不要な変数・インポート
   - モダンな構文への置き換え可能箇所

3. 簡略化の提案と、可能であれば修正を実行してください

## 簡略化の原則

- **可読性を維持**: 短くしすぎて読みにくくならないように
- **機能を保持**: 動作を変えない
- **段階的に適用**: 大きな変更は分割して提案

## 簡略化パターン

- 三項演算子の適切な使用
- アロー関数への変換
- 分割代入の活用
- スプレッド構文の使用
- Optional chaining (?.)
- Nullish coalescing (??)
- Array メソッド（map, filter, reduce）の活用
- 早期リターンパターン
- 重複コードの関数化

## 出力フォーマット

```
## コード簡略化レポート

### [ファイル名]

#### 変更 1: [簡潔な説明]

**Before:**
\`\`\`
// 変更前のコード
\`\`\`

**After:**
\`\`\`
// 変更後のコード
\`\`\`

**理由:** なぜこの変更が有効か

---

### 適用する変更を選択

上記の変更を適用しますか？
- [1] すべて適用
- [2] 個別に選択
- [3] 適用しない（レポートのみ）
```

## 使用方法

```
/simplify [ファイルパス]
```

引数なしで実行すると、最近変更されたファイルを自動検出します。
