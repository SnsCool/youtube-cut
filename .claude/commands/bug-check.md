# Bug Check コマンド

Codex CLI を使用してバグと脆弱性を検出するエージェントです。

## 実行手順

1. Bash ツールで以下のコマンドを実行してください：
   ```bash
   codex exec "このプロジェクトのコードを分析し、潜在的なバグ、エラー、セキュリティ脆弱性を特定してください。各問題について、ファイル名、行番号、重大度（Critical/Warning/Info）、説明を含めてください。"
   ```

2. 検出結果を重大度別に日本語で整理して報告してください。

## 重大度の定義

- **Critical**: セキュリティ脆弱性、データ損失の可能性、クラッシュを引き起こすバグ
- **Warning**: 潜在的なバグ、パフォーマンス問題、非推奨APIの使用
- **Info**: コードスメル、改善の余地がある箇所

## 出力フォーマット

```
## バグ・脆弱性検出レポート

### Critical（重大）
- [ファイル:行番号] 問題の説明
  原因: ...
  修正案: ...

### Warning（警告）
- [ファイル:行番号] 問題の説明
  原因: ...
  修正案: ...

### Info（情報）
- [ファイル:行番号] 問題の説明
  改善案: ...

---
検出数: Critical: X件 / Warning: Y件 / Info: Z件
```
